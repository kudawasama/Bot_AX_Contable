## Workflow: Implement (subagente)

1. Recibir instrucciones y targets del Orquestador.
2. Generar parche o archivos con cambios siguiendo convenciones.
3. Marcar nivel de riesgo y pasos para aplicación.
4. Entregar `patch`, `files`, `notes`, `risk`.

Criterios de éxito: parche aplicable y pasos claros para `git apply` o creación de archivos.
